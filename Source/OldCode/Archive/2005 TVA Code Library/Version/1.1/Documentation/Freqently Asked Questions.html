Why have a common code library?

 

Standards

          Standards in development make projects easier to maintain.  The developer of an application may not be the same person who ends up maintaining the code.

Speed

       The component model says: “It’s faster not to reinvent the wheel” (not to mention cheaper)

Reliability

       Reusing battle hardened tested code will produce more reliable applications.  This is critical in this type of environment.

Where do I get the library?

 

Reference binaries directly off the T: drive (\\opdat\opdat) in the " \\opdat\opdat\CtrlSys\CodeLibrary\Builds\Release\Current" folder – this will make sure your code is always referencing the very latest production build.  If you think something in the latest build has caused your code to break, the old builds will *always* be available in their respective folder.

 

How do I start using the library?

 

          •  Add designable components to your Visual Studio toolbox and start dragging desired components onto design surfaces.

          •  Reference desired assemblies in your projects directly off of T:\ drive and add the needed Imports statements to your code (see examples).

 

How do I add code to the library?

 

·        Create a test script for your code.  This doesn’t have to fancy, but it’s preferred if it works using the TVA testing library (\\opdat\opdat\CtrlSys\CodeLibrary\Common\Testing)

·        Send component description to the ESO Dot Net Reuse Committee (DNRC) e-mail group

·        If your code is approved for inclusion, the DNRC will give you a namespace assignment for your code or component

·        Place your code or component in the given namespace and rerun the test script to make sure your code still works after this change

·        Send code and test script to DNRC for library integration

·        That’s it!

How do I add custom settings to my app.config file?

 

Add reference to TVA.Config.dll

     

Imports TVA.Config.Common

 

    Public Sub New()

        MyBase.New()

 

        ' Example to make sure component based app.config settings exist before components are initialized

        Settings.Create("ServiceHelper.AboveNormalThreadPriority", False)

        Settings.Create("ServiceHelper.AutoLaunchMonitorApplication", False)

        Settings.Save() ' Only saves if modifications were made – this ensures config file is ready for component initialization

 

        ' This call is required by the Component Designer.

        InitializeComponent()

 

        ' Example to make sure general use app.config settings exist       

        Variables.Create("StatusLog.Template", ServiceHelper.AppPath & "StatusLog.template", VariableType.Text, "Define the status log template database", 1)

        Variables.Create("StatusLog.ConnectString", "Provider=Microsoft.Jet.OLEDB.4.0;Data Source=" & ServiceHelper.AppPath & "StatusLog.mdb", VariableType.Text, "Define the connection string for the local status log database", 2)

        Variables.Create("StatusLog.MaxEntries", 15000, VariableType.Int, "Define the maximum number of entries for the status log database", 3)

        Variables.Create("StatusLog.BackupsToKeep", 5, VariableType.Int, "Define the maximum number of status log database backups to keep", 4)

        Variables.Create("StatusLog.ExceptionThreshold", 10, VariableType.Int, "Maximum number of exceptions to tolerate before recreating the status log database", 5)

        Variables.Create("WebImages.Count", 5, VariableType.Int)

        Variables.Create("WebImages.URL0", "http://152.85.146.14/cgi-bin/wx_notes/tva_wx2.cgi?KCHA", VariableType.Text)

        Variables.Create("WebImages.URL1", "http://152.85.146.14/cgi-bin/wx_notes/tva_wx2.cgi?KBNA", VariableType.Text)

        Variables.Create("WebImages.URL2", "http://152.85.146.14/cgi-bin/wx_notes/tva_wx2.cgi?KHSV", VariableType.Text)

        Variables.Create("WebImages.URL3", "http://152.85.146.14/cgi-bin/wx_notes/tva_wx2.cgi?KMEM", VariableType.Text)

        Variables.Create("WebImages.URL4", "http://152.85.146.14/cgi-bin/wx_notes/tva_wx2.cgi?KTYS", VariableType.Text)

How can I easily save & restore a window’s location and size?

 

Add reference to TVA.Forms.dll

 

Imports TVA.Forms.Common

 

    Private Sub Form1_Load(ByVal sender As System.Object, ByVal e As System.EventArgs) Handles MyBase.Load

 

        RestoreWindowSettings(Me)

 

    End Sub

 

    Private Sub Form1_Closed(ByVal sender As Object, ByVal e As System.EventArgs) Handles MyBase.Closed

 

        SaveWindowSettings(Me)

 

    End Sub

 

Validation occurs in case settings are out of range with current screen resolution so that window is always visible.
